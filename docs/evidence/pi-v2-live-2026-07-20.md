# Pi v2 Live Evidence - 2026-07-20

## Scope

Pi 0.80.2 ran with GPT-5.6 Luna in a disposable Git repository through the
public girelay v2 lifecycle. Pi used an OpenAI-compatible local endpoint whose
credential was supplied only through the process environment. No real project
repository, remote, Pi settings, or credential file was modified.

## Integration Mechanism

Pi implements the Agent Skills standard and discovers user skills under
`~/.pi/agent/skills/`. girelay installed its protocol at
`~/.pi/agent/skills/girelay/SKILL.md` in an isolated temporary home.

Pi also supports TypeScript extensions, but this integration does not need one.
The girelay process already supplies the session environment and validates the
report command; the skill tells the model how to inspect, work, test, and report.
Keeping that boundary avoids duplicating Git or report logic in Pi-specific
code.

## Scenario

The reviewed fixture is in
[`examples/agent-live-fixture`](../../examples/agent-live-fixture/README.md).
Its baseline task-id normalizer failed two tests. The harness asked Pi to change
only `task_id.py`, run the complete unittest suite, follow the installed skill,
and submit a final semantic report.

## Observed Evidence

- girelay created a native `agent/pi-live` worktree from clean `main`.
- The baseline test suite failed before Pi ran.
- Pi ran inside the worktree with the durable intent and all adapter variables.
- Only `task_id.py` changed; the source checkout remained clean.
- `python3 -m unittest discover -s tests -v` passed both tests independently.
- Pi submitted a schema-valid, immutable, session-bound semantic report.
- The reported test command and result matched the harness's observed test.
- Read-only merge preview identified only `task_id.py` and planned both refs.
- Squash merge created task and source rollback refs, both verified by Git.
- Recovery inventory contained session snapshots and both merge rollback points.
- Cleanup removed the worktree and retained `agent/pi-live`.
- A post-run scan found no API credential in the preserved evidence artifacts.

## Report Review

Pi reported completed behavior, no remaining work or blockers, one design
decision, the exact passing test command, one Unicode-related risk, and a
concrete next action. These remain `reported-by-agent`. The harness separately
verified the changed path, test result, merge, refs, source cleanliness, and
cleanup state.

## Boundaries

- This fixture proves lifecycle compatibility, not model quality on large
  repositories.
- Pi extensions, interactive sessions, image inputs, and alternate providers
  were not tested.
- Raw model logs, credentials, temporary paths, and unrelated local content are
  not published.
- Authenticated model runs remain opt-in and are not a CI dependency.
